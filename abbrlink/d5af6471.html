关于PyAutoGUI的学习研究(二)

📢消息框功能

📢alert()函数

显示一个带有 文本 和 单个 OK 按钮 的简单消息框. 返回单击的按钮的文本.

📢confirm()函数

显示带有 确定 和 取消按钮 的消息框. 可以 自定义按钮的数量和文本.
返回单击的按钮的文本.

📢prompt()函数

显示带有 文本输入 和 确定 和 取消按钮 的消息框
返回输入的文本，如果单击取消，则返回 None。

📢password()函数

显示带有 文本输入 和 确定 和 取消按钮 的消息框. 键入的字符显示为 *.
返回输入的文本，如果单击取消，则返回 None

🧩截图功能

PyAutoGUI 可以截取屏幕截图，将它们保存到文件中，并在屏幕内定位图像。如果您有一个需要单击的按钮并希望在屏幕上找到它的小图像，这将非常有用。这些功能由随 PyAutoGUI 安装的 PyScreeze 模块提供。

屏幕截图功能需要 Pillow 模块。OS X 使用操作系统 screencapture 附带的命令。Linux 使用该 scrot命令，可以通过运行 sudo apt-get install scrot

🧩screenshot()函数

调用 screenshot() 将返回一个 Image 对象（有关详细信息，请参阅 Pillow 或 PIL 模块文档）。传递文件名字符串会将屏幕截图保存到文件并将其作为 Image 对象返回

在 1920 x 1080 屏幕上，该screenshot()函数大约需要 100 毫秒

如果您不想要整个屏幕的屏幕截图，还有一个可选的关键字参数 region, 你可以传递一个区域的左、上、宽、高的四整数元组来捕获.

在 1920 x 1080 屏幕上，定位函数调用大约需要 1 或 2 秒。这对于动作视频游戏来说可能太慢了，但适用于大多数目的和应用程序。

有几个“locate”功能。他们都开始看屏幕（或图像）的左上角，然后向右看，然后向下看。参数可以是

  1. locateOnScreen(image, grayscale=False) - 返回 image 屏幕上第一个找到的实例的（左、上、宽、高）坐标. 如果在屏幕上找不到, 则引发 ImageNotFoundException.

  2. locateCenterOnScreen(image, grayscale=False) - 返回 image 屏幕上第一个找到的 实例中心的 (x, y) 坐标. 如果在屏幕上找不到, 则引发 ImageNotFoundException.

  3. locateAllOnScreen(image, grayscale=False) - 返回一个生成器, 该生成器为在屏幕上找到图像的位置生成（左、上、宽、高）元组.
  4. locate(needleImage, haystackImage, grayscale=False) - 返回 haystackImage 中第一个找到的 needleImage 实例的（左、上、宽、高）坐标. 如果在屏幕上找不到, 则引发ImageNotFoundException.
  5. locateAll(needleImage, haystackImage, grayscale=False) - 返回一个生成器，该生成器为在 haystackImage 中找到 needleImage 的位置生成（左、上、宽、高）元组
locateAll、locateAllOnScreen 函数可用于 for 循环

也可用于list()

These “locate” functions are fairly expensive; they can take a full second to run. The best way to speed them up is to pass a region argument (a 4-integer tuple of (left, top, width, height)) to only search a smaller region of the screen instead of the full screen:

🧩灰度匹配

您可以传递grayscale=True给 locate 函数以提供轻微的加速（大约 30% 左右）。这会降低图像和屏幕截图的颜色饱和度，加快定位速度，但可能会导致误报匹配

🧩像素匹配

要获取屏幕截图中像素的 RGB 颜色，请使用 Image 对象的 getpixel()方法

或者作为单个函数，调用 pixel() 函数，它是之前调用的包装器

如果您只需要验证单个像素是否与给定像素匹配，请调用该 pixelMatchesColor() 函数，将它所代表的颜色的 X 坐标、Y 坐标和 RGB 元组传递给它

可选的 tolerance关键字参数指定每个红色、绿色和蓝色值在仍然匹配时可以变化的程度
